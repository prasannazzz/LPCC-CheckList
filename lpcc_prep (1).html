📖 Quick Reference & Formula Sheet

FIRST Set Rules

If A → a…, then a ∈ FIRST(A)
If A → ε, then ε ∈ FIRST(A)
If A → B… and ε ∈ FIRST(B), add FIRST(rest)
Terminal t: FIRST(t) = {t}

FOLLOW Set Rules

$ ∈ FOLLOW(S) (start symbol)
If A → αBβ: FIRST(β)−{ε} ⊆ FOLLOW(B)
If β=ε or ε∈FIRST(β): FOLLOW(A) ⊆ FOLLOW(B)

LL(1) Table Construction

For A → α:
• For each a ∈ FIRST(α) → M[A,a] = A→α
• If ε∈FIRST(α): for each b ∈ FOLLOW(A) → M[A,b] = A→α
LL(1): no cell has >1 entry

Shift-Reduce Parsing

Shift: push next input onto stack
Reduce: pop RHS, push LHS
Accept: stack = S$, input = $
Error: no valid action

Operator Precedence

Relations: <· (shift), ·> (reduce), =· (equal prec.)
Build precedence table from grammar
Handle ε-productions: skip

LR Parsers

SLR: uses FOLLOW sets
LALR: merges LR(1) states with same core
LR(1): canonical, most powerful
Items: [A→α·β, a]

Three-Address Code Forms

Quadruple: (op, arg1, arg2, result)
Triple: (op, arg1, arg2) — result implicit
Indirect Triple: pointer array to triples
Temp vars: t1, t2, t3…

TAC Statements

x = y op z — binary
x = op y — unary
x = y — copy
goto L — unconditional
if x relop y goto L — conditional
param x, call p,n

SDD / SDT

S-attributed: only synthesized attributes; evaluated bottom-up
L-attributed: synthesized + inherited; evaluated left-to-right DFS
Annotated parse tree = parse tree + attribute values

Boolean TAC

Use truelist and falselist
backpatch(list, label) fills in jump targets
Short-circuit: don't evaluate both sides

Basic Block Construction

Leaders: first instr, target of goto, instr after goto
Basic block: max sequence from leader to next leader−1
Flow graph: nodes = BBs, edges = control flow

DAG for BB Optimization

Leaf: input vars/constants
Interior: operator nodes
CSE: if node exists, reuse it
Reorder: any topological order of DAG

Peephole Optimizations

• Redundant loads/stores elimination
• Algebraic simplification (x*1=x)
• Constant folding (2+3=5)
• Dead code elimination
• Replace slow ops (x*2 → x+x)

Loop Optimizations

Code motion: move loop-invariant code out
Induction variable: simplify i*4 → j+4
Strength reduction: replace * with +
Loop unrolling: reduce branch overhead

Machine-Indep. vs Dep.

Indep: CSE, dead code, constant folding, loop opt.
Dep: register allocation, instruction selection, peephole, pipeline scheduling

Two-Pass Assembler

Pass 1: build Symbol Table (label → address), calculate LC
Pass 2: generate machine code using ST
When needed: forward references

Assembler Tables

MOT: Machine Op Table (mnemonic → opcode)
ST: Symbol Table (name → value/type)
POT: Pseudo-Op Table
LC: Location Counter

Macro Processor Tables

MNT: Macro Name Table (name → MDT index)
MDT: Macro Definition Table (body lines)
KPDT: Keyword/Default Param. Table
Expansion: substitute actual args into MDT

Loader Types

Absolute: loads at fixed address
Relocating: adjusts addresses at load time
Direct Linking (DLL): resolves external refs, combines modules
ESD: External Symbol Dictionary

Compiler Phases

Lexical → Syntax → Semantic → ICG → Opt → Code Gen
Symbol table & error handler span all phases

LEX Conflict Resolution

Longest match: prefer longer token (if vs i)
First rule wins: keywords before identifiers
Default: print/echo unmatched characters

Input Buffering

Two-buffer scheme: each buffer ~4096 bytes
Sentinels: EOF at end of each buffer
Reduces I/O calls; handles tokens spanning buffers
Pointers: lexemeBegin, forward

Token Specification

RE → NFA: Thompson's construction
NFA → DFA: subset construction
DFA → min DFA: partition refinement
LEX automates NFA→DFA compilation
